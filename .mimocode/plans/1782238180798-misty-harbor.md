## Архитектура для #1

### Текущие проблемы
- N+1 queries (getLocations, getById)
- Нет кэширования на сервере
- Нет WebSocket для real-time
- Mobile не поддерживает offline-first

### Что нужно

1. **Redis кэш** для frequently used queries:
   - `dashboard.kpis` (кэш 30 сек)
   - `agent.getLocations` (кэш 10 сек)
   - `shop.list` (кэш 60 сек)

2. **WebSocket** для real-time:
   - Обновление карты трекинга в реальном времени
   - Push уведомления о новых заказах
   - Статус синхронизации offline заказов

3. **Mobile offline-first**:
   - Realm/SQLite для локального хранения
   - Background sync при восстановлении сети
   - Conflict resolution стратегия

4. **API versioning**:
   - `/api/v1/trpc` для текущей версии
   - `/api/v2/trpc` для breaking changes
   - Миграция клиентов при обновлении
